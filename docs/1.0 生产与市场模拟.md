\section*{生产与市场模拟}

\textbf{1.0版本} 完成市场模拟系统。即完成市场模拟微分方程模型、确定商品类型与具体种类、接入建造拆除系统、搭建简易测试AI用来模拟市场参与者。要求搭建出自动扩大再生产直到利润归零的市场系统。

由于利润概念必须用到货币，故1.0上线临时货币系统，要求为1.2版本与2.2版本预留修改空间。由于消费需要消费者，需要为3.0劳动力系统做基础接入（即Pop系统）。

为了减轻模拟量，市场不采用直接模拟的形式，而是采用微分方程建模的形式模拟市场，增加生产即影响变量。对当前时间求解微分方程即可得到对应参数。

\textbf{任务：}
\begin{itemize}
    \item 搭建市场框架
    \item 完成微分方程建模与程序化
    \item 设计商品、生产者\&消费者、生产与消费
    \item 设计扩大再生产
\end{itemize}

从最简单的供需关系定律出发，带入二阶弹力系统得到市场变化模型
\begin{align*}
    D &= a - b \cdot P \\
    S &= c + d \cdot P
\end{align*}
其中$D$是需求，$S$是供应，$a$是标准需求量，$b$是价格抑制系数，$c$是标准供应量，$d$是价格供应系数。

得到过剩需求$E$
\[
E = D - S = (a-c) - (b+d)P
\]
当过剩需求$E$大于0时价格上涨，即价格加速度与$E$正相关，设其为正比关系，并考虑市场惯性
\[
m\ddot{p} + \rho\dot{p} = E = (a-c) - (b+d)P
\]
其中$m$是市场惯性，$\rho$是市场摩擦阻尼。设$m = k \cdot G$，其中$k$是惯性系数，$G$是总市场量。$\rho$是人为矫正量。

整理得到
\begin{align*}
    m\ddot{p} + \rho\dot{p} + (b+d)p &= a-c \\
    \ddot{p} + 2\zeta\omega_n \dot{p} + \omega_n^2 p &= f(t)
\end{align*}
其中$\omega_n$是市场周期，$\zeta$是阻尼，$f(t)$是外力。

得到三个解耦变化量，可以相互独立调整：
\begin{itemize}
    \item 市场周期$\omega_n$，用于调节震荡周期与类型，可通过改变供需弹性总和$(b+d)$或市场流动性$(m)$调节。调大$b+d$（如引入价格敏感型消费者），则周期缩短，波动变快；调大$m$（如增加交易手续费），则周期变长，市场变迟钝。
    \item 阻尼$\zeta$，用于调节市场恢复平衡的速度，可通过改变市场摩擦系数$(\rho)$调节。调大$\rho$（如加强涨跌停板），则波动快速收敛，趋于平稳；调小$\rho$（如允许高频量化），则波动呈持续震荡或发散。
    \item 外力$f(t)$，$f(t) = (a-c)/m + F/m$，其中$F(t)$是突发变量。外力用来调整弹力系统整体状态，通过改变均衡差价或施加突发变量均可调节此项。调节$a-c$可平移价格的最终锚定位置；随意注入$F(t)$（方波/脉冲）可模拟降息、关税等突发事件。
\end{itemize}

此处引入“供给滞后”概念，即“价格上涨导致供给扩大”存在生产周期$\tau$。则有供给函数变为时滞形式：$S = c + d \cdot P(t-\tau)$

代入方程后，系统变为时滞微分方程
\[
m\ddot{p} + \rho\dot{p} + b p(t) + d p(t-\tau) = a - c + F(t)
\]
注意到，由于设计中生产的扩大是由一系列离散事件（生产建筑建成）达成的，并不需要使用DDE的概念也可以表达出市场扩大再生产。

实际上只需要用到前后两种状态的ODE就可以做到上述效果。由于扩大再生产是一个离散的过程，即扩大一个单位生产建筑，扩大对应单位的商品供应。只需要在建筑完成的瞬间重新计算ODE即可。

此时ODE退化为
\[
m\ddot{p} + \rho\dot{p} + b p = a - S_0
\]
其中$S_0$表稳定的离散供给量。此时驱动力为$a - S_0$，表需求持续大于供给；回复力$b p$表示价格抑制需求。

此模型以欠阻尼形式存在，价格经过冲高随即迅速回落，最终围绕新的高均衡价值震荡。

这个过程中为了避免市场与供需关系解耦，现在需要对公式中各部分参数进行定义：
\[
m\ddot{p} + \rho\dot{p} + b p = a - S_0
\]
其中$a$是标准需求量，
\[
a = k \cdot n
\]
其中$k$是消费倾向系数，$n$是人口数量（Pop），消费倾向系数表后列。

其中$S_0$是当前供应量，
\[
S_0 = \frac{l}{L} \cdot K \cdot N
\]
其中$l$是劳动力数量，$L$是可容纳劳动力数量，$K$是单位建筑生产量，$N$是建筑数量。

其中$m$是市场惯性，$\rho$是市场摩擦阻尼。设$m = k \cdot G$，其中$k$是惯性系数，$G$是总市场量。$\rho$是人为矫正量，在测试过程中调整，此处设为零。

其中$b$是价格抑制系数，价格抑制系数表后列。

在确认模型后，下一步是确认市场中流通的商品类型与种类。

首先，商品根据大的性质可以广泛的分为：原材料、工业品、消费品、不动产/服务。

然后根据消费品的属性分为必需品和非必须品，其中必需品包括：食物、基本穿着；非必需品如加工食品、高档服装等。

最后选定1.0版本加入的商品为：谷物、加工食品、织物、服装、高档服装、煤炭、铁、钢、工具、住房、建造力。

每周期执行：所有建筑在利润率超过10\%的时候会自动扩建，每超过5\%就会多扩建1单位，扩建数量不能超过当前数量的10\%，计算向上取整。建筑扩建是产生了一个临时的生产单位，即投入最高30建造力每周期，在总建造力达到建造成本后该单位消失，对应建筑数量增加。

注意：为了模拟建造过程，此处将最大建造力/每建筑设置为30建造力每周期，故而需要一个栈结构——建造队列，用于记录建筑扩建的顺序，其中总建造力按顺序填满顺延。出栈的要求是已投入总建造力大于所需投入建造力。为了优先增大扩大再生产的速度，当建造队列完成时间超过52周期时，自动扩建建造部门，并且将建造部门扩建置顶。

当前1.0版本尚未有地区的概念，但耕地/原材料无限扩张是不科学的。设置耕地上限1w（即谷物农场与棉花种植园总数不得超过1w，达到后也不会扩大再生产），煤矿铁矿各500。建造部门不能超过1000级。其中没有被利用的土地每一级对应一级自给农场（自给农场不发工资）。

建造力所需成本应当从对应建筑现金池中扣除。每种建筑每级有现金池5k，该周期所得利润将存入现金池用于扩大再生产，再生产购买的建造力由现金池持续支付直到建筑建成。

当连续52周期雇佣人数不超过75\%时，建筑会每周期减少5\%等级直到雇佣人数超过75\%。

GDP计算方式为
\[
\text{GDP} = \text{市场商品流通量} \times \text{商品价格} + \text{建筑现金池}
\]

生产建筑需要雇佣劳动力才能进行生产。每个建筑每单位等级雇佣5k劳动力，其中75\%为劳工，设定工资5元，20\%为工程师，设定工资10元，5\%为资本家，设定工资为20元。当利润率低于0时，生产部门会每周期解雇5\%的劳动力，直到利润率回正。当利润率为正时，企业每周期尝试雇佣5\%劳动力。

劳动力的工资全部转化为人群的消费。

人群对于商品的消费基于自身的需求与商品满足需求的能力与商品的价格等因素。

则我们需先定义需求，

需求类型由消费组来决定，消费组参考维多利亚3包括：简朴衣物、粗制物品、基础食物、加热供暖、家用商品、标准衣物、服务、麻醉品、高档饮品、自由流动、高档食品、高档物品、休闲。

根据1.0版本选取的商品，1.0版本消费组有：简朴衣物、基础食物、标准衣物、住宅（注意：消费组只包括消费品，其他三种类型直接参与市场模型，而消费品被人群消费）。

而消费权重等于
\[
\text{消费权重} = \frac{\text{市场生产量} - \text{建筑生产消耗量}}{2} \times \text{市场价格}
\]
人群会优先购买消费权重较高的商品。

当人群对于必需品的消费未能满足时，人口死亡率会提升。即人口增长率由52周期5\%固定值转变为52每周期最高5\%，最低-20\%的变量。其中当满足75\%需求时增长率为0。1.0版本中必需品包括：简朴衣物与基础食物。
